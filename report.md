# Flappy Bird Training Report

## Architecture Overview
The model is a convolutional neural network (CNN) called FlappyBirdCNN. It includes:
- Three convolutional layers (32 channels each) with ReLU activations.
- A fully connected layer reducing 2,112 features to 512.
- An output layer producing two action values (flap or no flap).

## Hyperparameters
- Learning rate: 0.0001 (Adam optimizer)
- Epsilon (ε) for exploration: starts at 1.0, decays by 0.9975, minimum 0.01
- Discount factor (γ): 0.99
- Replay buffer size: 20,000
- Batch size: 90
- Training epochs: 50,000
- We also modified the score values: when passing a pipe, reward = 0.3 and penalising more when hitting one, reward = -10 (the rest of the rewards remained the same)

## Image Preprocessing & Max Pooling
To prepare the frames for the CNN, the following steps are applied:
1. Cropping: Remove irrelevant portions, such as the bottom area and some of the left side.  
2. Grayscale Conversion: Convert the cropped frame to a single-channel format.  
3. Sobel Edge Detection: Use Sobel filters in the x and y directions to highlight edges.  
4. Max Pooling: Downsample the edge-detected image, reducing noise and focusing on key features.  
5. Thresholding: Convert to a binary image for clarity and consistency across frames.

## Training function and Q-Learning Usage
The agent updates its Q-values by storing transitions in a replay buffer. During training, the network sees the current frame along with two prior frames to form a more informative 3-channel state. By sampling batches from the replay buffer, the agent learns from various past experiences. Epsilon-greedy exploration ensures a balance between trying new actions and exploiting known good ones. A target network is periodically synchronized to stabilize updates, preventing large fluctuations in the Q-values as learning progresses.
